 `n`n`n Wireframe Design Prompt `n`n Design a simple web application for a DIY Rug Tufting Workshop registration system for a small studio ( Rug Circle ). `n`n The objective is to allow users to: `n`n Register for a workshop `n`n Make payment using a dynamically generated UPI QR code via HDFC Vyapar `n`n Select a rug design they will create during the workshop `n`n Will be deployed on register.rugcircle.com `n`n The experience should be simple, mobile-friendly, and step-by-step , suitable for users with no technical background. `n`n `n `n`n`n Core Concept `n`n The system revolves around Campaigns . `n`n A Campaign represents a specific workshop event or promotional offering, such as: `n`n Workshop in a different city (example: Vadodara, Bhavnagar) `n`n Seasonal promotions (Valentine’s Day, Women’s Day, etc.) `n`n Special discounted workshop `n`n Partner/couple workshop event `n`n Every registration must be associated with exactly one Campaign . `n`n Campaign configuration is managed in the backend and defines: `n`n Campaign name `n`n Location `n`n Date `n`n Time `n`n Workshop price `n`n HDFC Vyapar QR payment configuration `n`n Available rug design options `n`n Campaign status (Active / Inactive) `n`n `n `n`n`n User Flow (Frontend Wireframes) `n`n`n Screen 1 – Campaign Landing Page `n`n Purpose: Provide workshop details and allow user to start registration. `n`n Elements: `n`n Campaign title `n`n Short workshop description `n`n Date and time `n`n Location `n`n Price `n`n Workshop highlights: `n`n Beginner friendly `n`n Materials included `n`n Take rug home same day `n`n Call-to-action button: `n`n Register for Workshop `n`n `n `n`n`n Screen 2 – Registration Form `n`n Purpose: Collect participant information. `n`n Fields: `n`n First Name `n`n Last Name `n`n Email Address `n`n Mobile Number `n`n Address (optional – used if rug delivery is needed) `n`n Button: `n`n Continue to Payment `n`n `n `n`n`n Screen 3 – Payment Page `n`n Purpose: Allow the user to complete payment using a dynamically generated UPI QR code via HDFC Vyapar integration . `n`n No screenshot upload is required. `n`n `n `n`n`n Elements `n`n Payment Instructions Clear instructions guiding the user to complete payment using UPI. `n`n Dynamic QR Code `n`n QR code generated via HDFC Vyapar `n`n QR code contains payment amount configured in the campaign `n`n QR is linked to the campaign and user session `n`n Amount to Pay Display workshop fee from campaign settings. `n`n Example: ₹2000 `n`n `n `n`n`n Payment Steps (UI guidance text) `n`n Scan QR code using any UPI app (GPay, PhonePe, Paytm, BHIM, etc.) `n`n Pay the exact amount shown `n`n Wait for automatic payment confirmation `n`n `n `n`n`n System Behaviour `n`n System listens for payment confirmation from HDFC Vyapar API/webhook `n`n Once payment is confirmed: `n`n Registration status is marked as Paid `n`n User can proceed automatically `n`n Show loading indicator while checking payment status `n`n Auto-refresh payment status periodically (example every 5–10 seconds) `n`n `n `n`n`n Button `n`n Check Payment Status `n`n OR automatically enable: `n`n Continue `n`n after successful payment confirmation. `n`n `n `n`n`n Optional UX Enhancements `n`n Show QR expiry timer (example 15 minutes) `n`n Show confirmation message: `n`n Payment received successfully `n`n `n `n`n`n Screen 4 – Rug Design Selection `n`n Purpose: Allow participants to select the rug design they will create during the workshop. `n`n Elements: `n`n Grid layout of predefined rug design images `n`n Each design selectable `n`n Simple visual layout suitable for easy selection `n`n Highlight selected design `n`n Optional: `n`n Show difficulty level (Easy / Medium) `n`n Show size indication `n`n Button: `n`n Submit Registration `n`n `n `n`n`n Screen 5 – Confirmation Page `n`n Purpose: Confirm successful registration and provide summary. `n`n Elements: `n`n Thank you message `n`n Registration details summary: `n`n Participant name `n`n Campaign name `n`n Date and time `n`n Location `n`n Payment status (Paid) `n`n Selected rug design preview `n`n Support contact information: `n`n Phone / WhatsApp number `n`n Optional message: `n`n We look forward to seeing you at the workshop. `n`n `n `n`n`n Backend Requirements (Admin Wireframes) `n`n`n Campaign Management Panel `n`n Admin should be able to create and manage campaigns. `n`n Campaign fields: `n`n Campaign Name `n`n Description `n`n Location `n`n Date `n`n Time `n`n Price `n`n HDFC Vyapar QR configuration / payment reference `n`n Upload rug design options `n`n Campaign status: `n`n Draft `n`n Active `n`n Closed `n`n Admin actions: `n`n Create campaign `n`n Edit campaign `n`n Activate / deactivate campaign `n`n Duplicate campaign `n`n `n `n`n`n Registration Management Panel `n`n Admin should be able to view participant registrations. `n`n Table view fields: `n`n Participant Name `n`n Email `n`n Mobile Number `n`n Campaign Name `n`n Payment Status `n`n Payment Reference ID (from HDFC Vyapar) `n`n Selected Rug Design `n`n Registration Date `n`n Filters: `n`n Filter by campaign `n`n Filter by payment status `n`n Filter by date `n`n Export options: `n`n CSV export `n`n Excel export `n`n `n `n`n`n Design Style Guidelines `n`n Clean and minimal UI `n`n Mobile-first layout `n`n Step-by-step flow `n`n Clear
